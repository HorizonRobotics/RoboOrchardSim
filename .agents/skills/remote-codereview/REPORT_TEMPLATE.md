# Remote Code Review Report Template

Use this template for:

- terminal review summaries
- top-level PR/MR summary comments
- any saved review report generated by this skill

Keep it concise. Report only validated, high-signal findings.

| Severity | Meaning |
|----------|---------|
| 🔴 High Risk | Must fix before merge — can cause incorrect behavior, outages, or security incidents |
| 🟡 Medium Risk | Should fix — meaningful correctness, maintainability, performance, or usability risk |

## Usage notes

- Merge duplicate findings before writing the final report.
- Keep severity to High Risk and Medium Risk only unless the skill is explicitly expanded to allow lower-priority findings.
- Do not include unvalidated suspicions.
- Do not include style-only or low-confidence findings.
- Keep the top-level summary aligned with any inline comments.
- When posting a no-issue summary comment, keep the `Findings` section to the exact required sentence above.
